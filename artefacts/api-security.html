Artefact: API Security Requirements Specification

Evaluation of API security requirements and a brief SRS for safe data sharing between Python and JSON/XML/SQL.

Overview

This artefact presents a Security Requirements Specification (SRS) evaluating the security considerations of a public REST API integrated with Python for data collection and sharing. It identifies key risks and provides mitigation requirements for secure communication, authentication, validation, storage, and governance when exporting data to JSON, XML, or SQL formats.

Security Requirements Specification (SRS)

Chosen API: OpenWeatherMap REST API (JSON over HTTPS).

Scope and Assets

  • Assets: API key, response payloads, SQL database, JSON/XML exports, logs.
  • Flow: Python client → HTTPS request → API response → validation → SQL store → JSON/XML export → logs.

Threat Model

  • Stolen API keys, MITM attacks, SQL/JSON/XML injection, insecure XML parsing (XXE), deserialisation, rate-limit abuse, dependency vulnerabilities, PII leakage.

Security Requirements & Mitigations

  • Transport: HTTPS only, certificate verification, request timeouts and retries.
  • Authentication: API key stored securely in environment variables; rotation and least privilege.
  • Validation: Treat all input as untrusted; validate fields and disable XML external entities.
  • Storage: Parameterised SQL; encrypt database and backups; redact sensitive exports.
  • Rate limiting: Implement client-side backoff and caching.
  • Logging: Log status codes and errors but exclude keys or PII.
  • Privacy: Data minimisation and retention policy for raw data.
  • Dependencies: Use pinned versions; perform vulnerability scans.

Secure Python Patterns

# Safe API call
import os, requests
from requests.adapters import HTTPAdapter, Retry

API_KEY = os.environ["OWM_API_KEY"]
BASE = "https://api.openweathermap.org/data/2.5/weather"
s = requests.Session()
retries = Retry(total=3, backoff_factor=0.5, status_forcelist=[429,500,502,503,504])
s.mount("https://", HTTPAdapter(max_retries=retries))
resp = s.get(BASE, params={"q":"Cardiff,GB","appid":API_KEY}, timeout=(3.05,10))
resp.raise_for_status()
data = resp.json()

# Safe SQL insert
import sqlite3
conn = sqlite3.connect("weather.db")
cur = conn.cursor()
cur.execute("INSERT INTO weather(city,temp_c) VALUES (?,?)",
            (data["name"], data["main"]["temp"]))
conn.commit()
